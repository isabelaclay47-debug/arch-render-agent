# 提示词助手页 + 拖拽上传 + 开机自启 — 设计

日期：2026-07-07
分支：`feat/prompt-helper`

## 背景与目标

现有网站的"AI 改提示词"依赖 ChatGPT 网页（导演对话），需要账号 + VPN。
很多目标用户**没有 ChatGPT 账号**、也**没有 VPN**，因此用不了主功能里的生图与提示词扩写。

本次交付三件事：

1. **提示词助手页**：拖图进来 → 识图 → 写一句想法 → 一键生成完整的中英专业提示词 →
   复制走（拿到任何免费生图工具里用）。**在没账号、没 VPN 时也能独立工作。**
2. **拖拽上传**：全站图片输入框支持把文件直接拖进来，不再只能点击选择。
3. **开机自启（手动开关）**：把常驻守护进程注册到系统登录启动，供部署者**按需**开启。

### 明确的边界（VPN）

主功能"生图"本质上必须走 ChatGPT（`chatgpt.com`），仍需 VPN + 账号，本次不改。
"没 VPN 也能用"**只保证提示词助手页**——它的本地路径完全离线。

## 一、提示词助手页 `/helper`

### 双引擎，默认 ChatGPT，可切本地

| 引擎 | 走哪条路 | 识图能力 | 提示词质量 | 何时默认 |
|---|---|---|---|---|
| **ChatGPT**（默认） | 服务器借导演对话看图 + 扩写 | 专业级，能认材质细节 | 最高 | 检测到 ChatGPT ready 时 |
| **本地**（可切换） | 浏览器模型识图 + 服务器本地拼装 | 详细画面描述（够用打底）| 良好 | ChatGPT 不可用时自动切；也可手动切 |

**交互流程：**

1. 页面加载即检测 ChatGPT（`/api/chrome_status`）：ready → 默认引擎 = **ChatGPT**；
   不可用 → 自动切到**本地**引擎（无需账号/VPN 也能用）。
2. 顶部给一个**引擎切换**开关：「ChatGPT（专业级 · 需账号+VPN）」/「本地（免账号/免 VPN）」，
   用户可随时手动切到本地。切到本地时才**懒加载** Florence 模型（ChatGPT 用户不必背 1.5GB）。
3. 用户拖图进来 + 写一句想法 + 勾选储备库模块（复用主页 `PROMPT_LIBRARY`）。
4. 点「生成提示词」，按当前引擎：
   - **ChatGPT** → `POST /api/helper_refine`：把图 + 想法 + 勾选项发给导演对话，看图后按三段格式扩写，
     返回专业版中英提示词。要求无进行中的渲染，否则 409（见下）。
   - **本地** → 浏览器 Florence-2 出「详细描述」→ `POST /api/helper_build` 本地拼装。全程离线。
5. 结果中英双语展示 + 一键复制。

### 本地识图（浏览器内，离线）

- 模型：**Florence-2-large**（`onnx-community` / `Xenova` ONNX），任务 `<MORE_DETAILED_CAPTION>`。
- 运行库：`transformers.js`，优先 **WebGPU**，不支持则退 **wasm**。
- **离线保证**：`transformers.js` 运行库、wasm 和模型权重都由本地 Flask 提供
  （`/vendor/transformers/*`、`/models/florence2-large/*`），页面运行时**不请求任何外部域名** →
  不需要 VPN。首次安装时由启动脚本下载一次这些资产（约 1.5GB），之后完全离线。
- **首次下载也要免 VPN**：模型资产从**国内可直连的镜像**拉取（首选 ModelScope，其次 `hf-mirror.com`），
  不走 `huggingface.co`，确保没有 VPN 的部署者也能装成功。下载脚本内置校验（大小/哈希），
  失败可断点重试；也支持"离线包"分发（把 `models/` 目录随发布包一起给）。
- 诚实边界：本地模型给出的是**详细场景描述**（结构 / 大致材质 / 光线时段 / 构图 / 风格氛围），
  **不保证**准确说出专业材质名（清水混凝土 / 焦杉木 / 耐候钢等）。专业级细节仍以 ChatGPT 路径为准。
- 健壮性（非新功能，属错误处理）：模型加载失败 / 设备太弱时，页面提示"本机无法本地识图"，
  仍允许用户手动描述画面后继续拼装——不因识图失败而卡死。

### 本地拼装（服务器，离线）

在 `prompt_engine.py` 新增**确定性**拼装函数（无 LLM、不联网），复用现有专业知识：

```
build_prompt_locally(intent: str, image_desc: str, presets: list[str]) -> dict
    返回 {understanding_zh, prompt_zh, prompt_en}
```

- 按 `PROMPT_SKELETON` 的顺序，把「用户想法 + 识图描述 + 勾选的储备库文本」组织成结构化段落。
- 中文提示词：给人读；英文提示词：末尾强制附加 `GENERATION_BASICS`（和主功能同一套底线）。
- `understanding_zh`：一句话复述"我据此理解你要什么"，让用户核对。
- 这是"结构化专业扩写"，不是对话式改写——和自动生成**共用同一套知识底座**。

### 新增后端接口（`app.py`）

- `GET /helper` — 渲染助手页。
- `POST /api/helper_build` — 入参 `{intent, image_desc, presets[]}`，出双语提示词。纯本地，不联网。
- `POST /api/helper_refine` — 入参 `{image(上传), draft_prompt}`，用导演对话精修。
  - **前置条件**：`chrome_status` = ready 且主渲染 `S["state"]` ∈ {idle, done, error}（无进行中的渲染）。
  - 否则返回 409 + 提示"当前正在渲染，稍后再用精修"。
  - 实现：开一个**独立的、一次性的** `ChatGPTClient`（只用导演页，不生图），
    发「图 + 草稿 + 要求按三段格式重写」，解析 `parse_director_reply`，返回后关闭。避免与主会话状态耦合。
- 静态资产路由：`/vendor/transformers/<path>`、`/models/florence2-large/<path>`（从本地目录发送）。

## 二、拖拽上传（全站）

- 给主页的「原图」「意向图」两个 `.drop` 以及助手页的图片区，加 `dragover/dragleave/drop` 事件：
  拖入文件时写进对应 `<input type=file>`（或助手页的内存图），并触发现有 `preview()`。
- 视觉反馈：拖到框上时高亮边框（复用现有 `:hover` 风格）。
- 纯前端改动，不动后端。

## 三、开机自启（手动开关）

- 新增 `开机自启-开.bat`：用 `schtasks` 注册一个"登录时运行"的计划任务，
  命令 = 以 `pythonw` 启动 `supervisor.py`（沿用现有守护逻辑）。**默认不注册，双击才生效。**
- 新增 `开机自启-关.bat`：删除该计划任务。
- macOS 等价：`开机自启-开-Mac.command` / `开机自启-关-Mac.command`，用 `launchd`（`~/Library/LaunchAgents` plist 的 load/unload）。
- 注册后：部署者重启电脑，服务在后台自起待命；专用 Chrome 登录一次后即可用。

## 数据流小结

```
页面加载 → 检测 ChatGPT →  ready? ──默认──→ [ChatGPT 引擎]
                              │                图+想法+勾选 → /api/helper_refine → 导演对话看图扩写 → 专业版中英提示词 → 复制
                        不可用/手动切
                              ↓
                        [本地引擎]  拖图 → 浏览器 Florence-2 出描述 ┐
                        用户想法 + 储备库勾选 ──────────────────────┼→ /api/helper_build → 本地拼装 → 中英提示词 → 复制
```

## 不做（YAGNI）

- 不做助手页的图片**生成**（目标用户没账号，本就不生图；助手只产出"提示词"）。
- 不做本地大模型（Ollama 等）—— 依赖过重，不适合非技术用户分发。
- 不改主生图流水线的 VPN 依赖。
- 开机自启不做默认开启、不做开机弹窗。

## 验收标准

1. 断网（模拟无 VPN、无外网）+ 手动切「本地」引擎打开 `/helper`：能拖图、本地出描述、生成中英提示词、复制成功。
2. ChatGPT ready 时，`/helper` **默认** ChatGPT 引擎，点「生成」返回专业版三段提示词；可手动切「本地」。
   ChatGPT 不可用时，页面自动落到「本地」引擎、不报错。
3. 主渲染进行中调用 `/api/helper_refine` 返回 409，不干扰主会话。
4. 主页原图/意向图、助手页图片区都能拖拽上传。
5. `开机自启-开.bat` 注册计划任务、重启后守护自起；`开机自启-关.bat` 干净移除。
6. `helper_refine` 的图片上传走 `_resolve`/临时目录，无路径穿越。
