# 建筑渲染智能体

用你的 ChatGPT 套餐额度，自动完成"提示词组织 → 生图 → 对比查篡改 → 迭代优化"的建筑渲染循环。

## 使用步骤

1. **双击 `start_chrome.bat`** — 弹出一个专用 Chrome 窗口，在里面登录 chatgpt.com（只需登录一次，之后会记住）。这个窗口在整个渲染过程中**不要关**，可以最小化。
2. **双击 `run.bat`** — 自动打开操作页面 http://127.0.0.1:5001
3. 在页面上：填需求描述、上传**原图**（必传）、上传**意向图**（可选多张）→ 点"开始渲染循环"。
4. 生图前 AI 会先判断你的需求是否明确：**有影响出图方向的疑问会先反问你**（页面上弹出提问卡，不消耗生图额度），答清楚才开始生图；点评有歧义时同样会反问。
5. 程序自动循环：AI 把你的想法扩写成全面提示词 → ChatGPT 生图（可选画质/画幅）→ 与原图对比检查有没有篡改/瞎改/文字变形 → 判断下一步是「在上一张基础上精修」还是「从原图重画」。**每几张暂停一次**（在界面上用「几张一停」设定，默认 1 张一停，最省额度、能最早纠偏），你看过程图、做三选一：
   - 写点评后点"按点评继续" → 继续下一批；
   - 点任意过程图下的"🖌 圈选局部修改" → 用**画笔或框选**在图上标红要改的区域＋写修改指令 → AI 被严格要求**只改圈出的部分、不动周边、保持文字清晰**，改完回到点评暂停，可连续多次局部修改（后续迭代会以修改结果为新底图）；
   - 点"满意，输出到桌面" → 最终图保存到桌面（文件名 `渲染结果_月日_时分.png`）。
   - 任何时候也可以点"提前结束"直接拿当前最新图。
6. 所有过程图和提示词记录保存在 `workspace\会话时间戳\` 里，可回溯。

## 工作原理

- 一个"导演对话"（ChatGPT 会话）全程负责：理解需求 → 按固定的**八段式建筑渲染提示词骨架**（任务声明/建筑本体/视角构图/光线时段/材质细节/环境配景/风格氛围/相机质感 + 五条忠实性硬约束）组织提示词 → 每轮拿到生成图后与原图逐项对比（形体/层数/开窗/视角/特殊构件）→ 修订提示词。
- 生图走**双路径**，兼顾省额度和忠实性：
  - **精修上一张**（默认，针对局部瑕疵）：把上一张生成图喂回去做增量修改，只改 QC 指出的缺陷、其余逐像素保留——缺陷能真正被改掉，不用整张重掷骰子，省生图额度。
  - **从原图重画**（针对形体被明显篡改或需要全局大改）：回到原图按修订后的完整提示词重画，纠正累积漂移。
  - QC 每轮都拿新图和**原图**对比来判断走哪条路；一旦发现偏离原图就自动回退重画校准。
- 每一版发给生图模型的指令都在代码层强制附带基本底线（不改建筑细节、画质清晰、文字端正、画幅比例按选项处理），不依赖模型"记得写"。
- 你的点评是最高优先级修改依据，会逐条落实进提示词。

## 常见问题

- **"连不上 Chrome 调试端口"** — 没运行 `start_chrome.bat`，或专用 Chrome 被关了。
- **"找不到输入框"** — 那个 Chrome 里没登录 chatgpt.com，或弹了人机验证，去处理一下。
- **生图很慢/失败** — ChatGPT 生图本身要 1-3 分钟/张；套餐额度用尽时会一直等不到图，去 Chrome 里确认。
- **界面改版导致选择器失效** — 修改 `chatgpt_client.py` 顶部的 `SEL` 字典即可。

## 依赖

Python 3 + `flask` + `playwright`（见 `requirements.txt`）。Playwright 只接管现有 Chrome，无需下载浏览器内核。
